# Quiz Funnel Runner — MVP

Бот для автоматического прохождения и анализа квиз-воронок (онлайн-опросников). Скрипт проходит воронку от первого шага до экрана оплаты (paywall), делает скриншоты каждого шага и классифицирует экраны с помощью эвристик.

## Особенности
- **Мобильная эмуляция:** Работает через Playwright с профилем iPhone 13.
- **Автоматическое прохождение:** Сам закрывает попапы, принимает cookies, выбирает ответы, заполняет текстовые поля (рост, вес, возраст, имя) и email.
- **Интеллектуальная классификация:** Распознает 6 типов экранов (`question`, `info`, `input`, `email`, `paywall`, `other`). Логика основана на анализе структуры интерфейса (подсчет интерактивных элементов, наличие пикеров), а не на жестких текстовых привязках.
- **Параллельное выполнение:** Поддерживает одновременный запуск нескольких воронок для экономии времени.
- **Остановка на пейволле:** Завершает работу и делает финальный скриншот при обнаружении страницы с тарифами/подпиской.

## Требования
- Python 3.9+
- Установленные браузеры для Playwright

## Установка

1. Клонируйте репозиторий:
```bash
git clone <ссылка_на_репозиторий>
cd quiz-funnel-runner-mvp
```

2. Создайте виртуальное окружение и активируйте его (рекомендуется):
```bash
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Mac/Linux:
source venv/bin/activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Установите браузеры Playwright:
```bash
playwright install chromium
```

## Настройка

Конфигурация находится в файле `config.json`:
- `funnels`: список URL-адресов воронок для прохождения.
- `max_funnels`: ограничение на количество обрабатываемых ссылок.
- `headless`: `true` для фонового режима, `false` для визуального отображения работы браузера.
- `max_steps`: защита от зацикливания (максимальное количество шагов).
- `slow_mo_ms`: задержка между действиями (рекомендуется 50ms для стабильности).

## Запуск

**Стандартный (последовательный) запуск:**
```bash
python main.py
```

**Параллельный запуск (быстрее, проходит все ссылки из конфига одновременно):**
```bash
python main.py --parallel
```

### Дополнительные опции
Вы можете переопределить настройки из командной строки:
```bash
python main.py --config custom_config.json --headless false
```

## Структура результатов

После успешного прогона в корне проекта появится папка `results/`.

Она разделена на две части:
1. **По воронкам:** `results/<slug_воронки>/`
   Здесь лежат скриншоты конкретной воронки в хронологическом порядке (например, `01_question.png`, `02_info.png`) и подробный лог прохождения `log.txt`.
   
2. **По типам (агрегировано):** `results/_classified/`
   Скриншоты со всех воронок, отсортированные по папкам классов: `/question`, `/info`, `/input`, `/email`, `/paywall`, `/other`.

Общая статистика прогона сохраняется в `results/summary.json`.

## Тестирование (Smoke Test)
В проекте предусмотрен минимальный предсдачный тест. Он проверяет наличие и корректность сгенерированных артефактов (проверяет `summary.json`, наличие логов и то, что бот действительно дошел до пейволла).

Запускать **строго после** полного прогона `main.py`:
```bash
python smoke_test.py
```
Ожидаемый результат: `SMOKE TEST PASSED`
